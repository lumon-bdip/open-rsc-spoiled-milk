# Contributor Guides

This folder explains how Spoiled Milk work should be organized before more
people are added to the project.

## Guides

- [Project owner workflow](project-owner-workflow.md): how the owner should
  use branches, plans, reviews, and releases.
- [Contributor workflow](contributor-workflow.md): how contributors should
  submit code, docs, ideas, or artwork.
- [Branch and pull request workflow](branching-and-prs.md): branch names,
  pull request rules, and merge expectations.
- [Plan document workflow](plan-doc-workflow.md): how plans move from proposal
  to active work to completed records.
- [Idea submissions](idea-submissions.md): how new gameplay or content ideas
  should be proposed.
- [Artwork submissions](artwork-submissions.md): how sprites and art should be
  submitted without automatically changing the game.
- [AI-assisted workflow](ai-assisted-workflow.md): rules for contributors using
  AI tools.
- [AI workspaces](../workspaces/README.md): separate worktree setup for running
  multiple AI/contributor sessions without mixing branches.
- [GitHub owner setup](github-owner-setup.md): recommended GitHub settings for
  protecting the project.

## Short Version

`main` is the official game. Branches are temporary workspaces. Plan documents
are the long-term source of truth. Pull requests are where work gets reviewed.

Contributors should add work through branches and pull requests. The owner
decides what moves from proposal to active work, what gets merged, and what is
included in a release.
