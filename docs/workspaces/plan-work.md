# Plan Work Workspace

Use this workspace for planning, audits, documentation, and code reading.

Allowed by default:

- Update `docs/`.
- Add or refine plan docs.
- Run read-only audits and status commands.
- Propose code changes without implementing them.

Avoid by default:

- Server launches.
- Release packaging.
- Large code edits.
- Gameplay/content commits unless the task is explicitly moved out of planning.

Before handing work back, summarize which docs changed and whether any plan
items became implementation-ready.
