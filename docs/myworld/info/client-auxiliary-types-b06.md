# B06 Client Auxiliary-Type Extraction

This branch addresses the client `javac -Xlint:all` auxiliary-class warning
family identified by the 2026-07-12 code-health audit. It changes source-file
ownership only: the moved types retain their package, package-private
visibility, fields, methods, and call sites.

## Extraction Scope

The published-main compiler report contained 310 auxiliary-class warning
occurrences across 24 types that were declared in one source file and used
from another:

| Area | Types | Prior warnings |
| --- | ---: | ---: |
| Interface callbacks (`ButtonHandler`) | 1 | 64 |
| Frame/input support (`Frame`, `FrameBufferPool`, `KeyBinding`) | 3 | 88 |
| Composite-scene data | 10 | 120 |
| Chunk/remaster renderer data | 10 | 38 |
| **Total** | **24** | **310** |

Each warning-producing type now resides in its correctly named source file.
`FrameBuffer` and `RemasterShadowIndoorFlood` moved with their closely coupled
frame and roof-coverage owners so that the new files did not introduce a
replacement cross-file warning. In total, the ownership test covers 26 moved
top-level types.

The composite-scene group comprises `CompositeWorldSpriteTexture`,
`EntitySpriteOccluderFaceStats`, `LegacyEntitySpriteDebugStats`,
`LegacyEntitySpriteDepthEvaluation`, `StaticWorldCommand`,
`StaticWorldCommandBuilder`, `StaticWorldMaterialTriangle`,
`WorldSpriteAnchorMatch`, `WorldSpriteCommand`, and `WorldSpriteMatch`.

The chunk/remaster group comprises `OpenGLWorldChunkDrawStats`,
`OpenGLWorldChunkUploadStats`, `ResidentChunkReadiness`,
`ShaderVertexParityStats`, `RemasterGlowMask`, `RemasterGlowMaskBuild`,
`RemasterShadowInventory`, `RemasterShadowRoofCoverage`,
`RemasterShadowMaskBuild`, and `RemasterTerrainShadowMask`.

## Deliberate Boundaries

- No type was made public and no package or call site changed.
- Owner-local implementation helpers that are not referenced by another
  source file do not produce the targeted compiler warning. They remain in
  place even where the report-only Checkstyle `OneTopLevelClass` rule notes
  them; extracting them would broaden this branch into a separate structural
  cleanup family.
- Nested interface handlers in the party and clan interfaces are correctly
  owner-scoped and remain unchanged.
- The committed javac baseline has no auxiliary-class fingerprints because
  B04 gates only selected correctness categories. The baseline therefore does
  not change; the complete client compiler report is the delta authority for
  this report-only family.
- Renderer behavior, input semantics, UI layout, and server code are outside
  this source-ownership change.

## Regression Coverage

`tests/myworld/test-client-auxiliary-types.py` guards the mapping between all
26 moved types and their source files. It also scans both client source roots
for any incorrectly named top-level type referenced by a different source and
executes package-local fixtures for the interface callback, key bindings, and
frame-buffer pool reuse/reset behavior.

Existing input and renderer source-ownership tests now include the extracted
files in their inspected subsystem. The renderer guardrail suite exercises
input modifiers, keyboard shortcuts, world geometry, material grouping,
shading diagnostics, roof visibility, relog handling, and frame capture.

As an additional behavior-preservation check, the branch and published-main
client were built separately. Disassembled method bytecode for the 29 class
files generated by the moved types (including generated nested classes) was
identical after ignoring only javac's source-filename label.
