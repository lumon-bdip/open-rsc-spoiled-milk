# Contributing to Spoiled Milk

Spoiled Milk is owner-directed. Contributions are welcome, but the project
owner has final say on what becomes official game content.

## Start Here

Read these guides before submitting work:

- [Contributor guide index](docs/contributor-guides/README.md)
- [Branch and pull request workflow](docs/contributor-guides/branching-and-prs.md)
- [Plan document workflow](docs/contributor-guides/plan-doc-workflow.md)
- [AI-assisted contribution rules](docs/contributor-guides/ai-assisted-workflow.md)
- [AI workspaces](docs/workspaces/README.md)

## Core Rules

- Do not push directly to `main`.
- Use a topic branch for each focused change.
- Open a pull request into `main`.
- Keep gameplay changes tied to a plan doc unless the change is a small bug fix.
- Submit ideas and artwork as submissions first. They are not automatically
  accepted as official game content.
- Do not overwrite unrelated files or revert work you did not create.
- Mention what you tested in the pull request.

## Branch Names

Use a clear prefix:

- `fix/short-description`
- `feat/short-description`
- `content/short-description`
- `balance/short-description`
- `art/short-description`
- `idea/short-description`
- `docs/short-description`

## Project Authority

`main` is the official source of the game. A merged pull request means the
owner accepted that work into the official project state.

An accepted idea or artwork submission only means the project is keeping it for
consideration. It does not mean it is approved for implementation.
